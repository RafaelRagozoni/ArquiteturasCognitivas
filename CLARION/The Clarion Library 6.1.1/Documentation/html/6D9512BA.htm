This is the base parameters class for component classes that derive from ClarionComponent

Namespace: Clarion.Framework.Core
Assembly: ClarionLibrary (in ClarionLibrary.dll) Version: 6.1.1.0 (6.1.1.0)

Syntax

public class ClarionComponentParameters

Remarks

This class contains all of the parameters that may be used by the ClarionComponent class. All classes that extend ClarionComponent should also contain their own parameters class that extends this class and adds any parameters that are used by those classes. This is how all of the "built-in" components within the Clarion Library have been implemented. It is HIGHLY recommended that users defining their own (custom) components follow this convention as well. Details on how to implement a parameters class for a custom component can be found in the Advanced Tutorial under the Tutorials section of the Clarion Library package.

Note

The parameters classes can be found throughout the Clarion Library whenever a class contains parameters that may be set by a user. This concept was implemented to allow for some "special capabilities" and also to clean-up/streamline parameter usage throughout the system. The parameters classes enable parameters to be adjusted both locally and globally. In general, global parameter changes are performed statically through the GlobalParameters property and local parameter changes are performed on single instances through the Parameters property.

Examples demonstrating how parameter adjustments may be performed using both methods can be found in the examples section.
Note
Global parameter settings are ONLY applicable prior to the initialization of an instance of a component. During the initialization process, the global parameter settings are used to set the "default" values of the local parameters for a component. Once initialized, the component will only use the local parameters. Therefore, global parameter changes SHOULD ALWAYS be performed BEFORE instances of a component are initialized.

Examples

Suppose you wanted to change the "ELIGIBILITY" parameter for a single instance of a class derived from ClarionComponent. This constitutes a "local" parameter change and is accomplished as follows:
CopyC#
SomeClarionComponentSubclass comp;

// Elided code containing the initialization of comp using the <see cref="T:Clarion.AgentInitializer" />

comp.Parameters.ELIGIBILITY = true;     //Note that ELIGIBILITY is already true by default, so this doesn't actually change anything.
 Now, suppose instead that you wanted to change the "ELIGIBILITY" parameter for all instances of all classes derived from ClarionComponent. This constitutes a "global" (or static, in programming terms) parameter change and is accomplished as follows:
CopyC#
ClarionComponent.GlobalParameters.ELIGIBILITY = false;

// Elided code containing the initialization of particular ClarionComponent instances (as was demonstrated above)
 Furthermore, suppose you wanted to change the "ELIGIBILITY" parameter for ONLY those components that are derived from a specific subclass of ClarionComponent. This can be accomplished in much the same way as was demonstrated above:
CopyC#
SomeClarionComponentSubclass.GlobalParameters.ELIGIBILITY = false;

// Elided code containing the initialization of particular ClarionComponent instances (as was demonstrated above)
Caution

Changing a parameter globally at a specific point on the inheritance hierarchy (as was demonstrated above) requires that the parameter in question be overridden (or hidden) at that point in the hierarchy. In other words, although all parameters defined within the base classes of a subclass of ClarionComponent are globally changeable at any point in the hierarchy, if the subclass does not override (or hide) a parameter, then ANY change made globally to that parameter will be applied at the level where that parameter was last overridden, regardless of the particular subclass in which the global parameter change is initiated.

For example, suppose there is a subclass of ClarionComponent called "SomeOtherClarionComponent" whose parameters class extends ClarionComponentParameters but does not override ELIGIBILITY. In this case, performing the following global parameter change:
CopyC#
SomeOtherClarionComponent.GlobalParameters.ELIGIBILITY = false;
 will actually change the "ELIGIBILITY" parameter at the ClarionComponentParameters level. In other words, the command above does the same thing as:
CopyC#
ClarionComponent.GlobalParameters.ELIGIBILITY = false;

Keep this in mind when you are performing global parameter changes as it could cause unintended consequences during runtime. For the "built-in" components in the Clarion Library, if a parameter was not overridden at a certain point in the inheritance hierarchy, then, in all likelihood, it is not used by that component. For example, Rule has a very specific method for calculating eligibility, and therefore does not make use of the ELIGIBILITY parameter. Making a change to Rule.GlobalParameters.ELIGIBILITY has no effect on how eligibility is calculated for rules. Instead, it actually changes the value of ClarionComponent.GlobalParameters.ELIGIBILITY, thus changing that value for ALL subclasses of ClarionComponent. As a consequence, other subclasses that do make use of the ELIGIBILITY parameter, may not work correctly due to this unintended parameter change.

As a general rule of thumb (when creating a custom component), if your component makes use of a parameter that was defined in a base class of that component, then you should override that parameter in your component's parameters class. This will allow users of your component to globally change that parameter at either the base class level or the level of your component. Details on how to correctly implement this can be found in the Advanced Tutorial under the Tutorials section of the Clarion Library package.
